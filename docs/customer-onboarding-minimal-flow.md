# Customer Onboarding Minimal Flow

## Zweck

Dieses Dokument beschreibt den minimalen spaeteren Onboarding-Pfad fuer eine neue Kunden-Domain.

## Prueffragen

1. Ist es doktrinkonform?
   Ja. Keine Domain wird ohne klare Bindung, Scope und Freigabe angebunden.
2. Bleibt es innerhalb des Workspace?
   Ja.
3. Hat es irgendeine externe Wirkung?
   Nein.
4. Braucht es Approval?
   Das Modell nicht. Jede spaetere Kundenanbindung ja.
5. Ist ein Rollback oder Rueckweg beschrieben?
   Ja. Ohne Vollstaendigkeit keine Freischaltung.

## Minimale Schritte

1. Domain identifizieren
2. Site-Rolle festlegen: Referenz oder Kunde
3. WordPress als Plattform bestaetigen
4. Minimal-Scope festlegen
5. Kanal festlegen
6. Rollback-Verantwortung festlegen
7. Lizenz, Policy und Rollback-Profil vorbereiten
8. ohne Vollstaendigkeit auf `approval_required` oder `observe_only` bleiben

## Status

- Onboarding-Modell: `blueprint_ready`
- echte Kundenanbindung: `approval_required`
