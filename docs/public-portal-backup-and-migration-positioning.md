# Public Portal Backup And Migration Positioning

## Zweck

Dieses Dokument beschreibt, wie Backup-, Restore- und Migrationsfaehigkeit zur oeffentlichen Produktpositionierung passt, ohne HA- oder Fulfillment-Versprechen zu machen.

## Oeffentliche Aussage

- die Suite ist fuer kontrollierten, restorable und spaeter migrierbaren Betrieb geplant
- Hetzner bleibt die zentrale Control Plane fuer den Start
- Restore- und Migrationsfaehigkeit bedeuten keine Live-HA oder Null-Ausfall-Garantie

## Nicht oeffentlich zu behaupten

- automatische Multi-Region-Verfuegbarkeit
- Null-Ausfall-Garantie
- horizontales Auto-Scaling

- Public Portal Backup and Migration Positioning: `blueprint_ready`
