# Public Portal Header Footer Harmonization

## Zweck

Dieses Dokument beschreibt die stabile Rahmennavigation des oeffentlichen Portals.

## Header

- Header zeigt nur oeffentliche Produktpfade
- priorisierte Navigation: Home, Explore, Features, Plugin, Security, Licensing, Buy, Downloads, Support, Docs
- keine Legal-, Operator- oder Customer-Funktion im Header

## Footer

- Footer zeigt dauerhaft die Legal-Navigation
- Footer spiegelt die Operator-Identitaet und den Support-Kontakt
- Footer erinnert daran, dass geschuetzte Routen nicht oeffentlich sind

## Doktrinbindung

- Header steigert Orientierung
- Footer sichert rechtliche und vertrauensbildende Klarheit
- weder Header noch Footer duerfen Protected Routes oeffentlich normalisieren

## Status

- Header/Footer Harmonization: `blueprint_ready`
