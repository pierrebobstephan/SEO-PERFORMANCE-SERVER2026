# Public Portal Offer Communication Model

## Zweck

Dieses Dokument beschreibt, wie das sichtbare oeffentliche Angebot kommuniziert wird, ohne geschuetzte Fulfillment-Schritte zu oeffnen.

## Prinzip

- sichtbare Terms-Aussagen duerfen wiederverwendet werden
- bestaetigte Preise und Paketstufen duerfen sichtbar genannt werden
- Buy- und Licensing-Seiten duerfen enthalten/nicht enthalten sauber erklaeren
- exact licensed domain, documentation und licensed download access bleiben die konstanten Kernelemente

## Nicht-Ziele

- kein Checkout
- kein Open Download
- keine License API
- keine implizite Sofortfreischaltung
