# Public Portal Open Fields Register

## Zweck

Dieses Register sammelt die offenen Felder, die im Portal weiterhin klar als unbestaetigt markiert werden muessen.

## Offene Felder

- finaler Support-Kontakt
- Support-E-Mail
- Commercial Inquiry Path
- Access-Request-Workflow
- Customer-Authentifizierungsmodell
- reale Lizenzfreischaltung
- reale Download-Ausgabe

## Markierungsregeln

- unbestaetigt: `operator input required`
- ungeklaert: `source not yet confirmed`
- technisch oder organisatorisch gesperrt: `approval_required`

## Portalpflicht

- kein offenes Feld darf durch Vermutung oder Marketing-Sprache kaschiert werden
- offene Felder duerfen conversion-seitig vorbereitet, aber nicht vorgespiegelt werden
