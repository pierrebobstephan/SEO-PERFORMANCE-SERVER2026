# Public Portal Status Components Model

## Zweck

Dieses Dokument beschreibt die sichtbare Statuskomponente fuer das oeffentliche Portal.

## Statusfelder

- `support_readiness_state`
- `commercial_readiness_state`
- `access_request_state`
- `legal_readiness_state`
- `download_readiness_state`

## Darstellung

- nur Status und Kontext
- keine Buttons mit echter Ausfuehrung
- keine Formulare
- keine Entitlement- oder Login-Simulation

## Haupteinsatz

- `/buy`
- `/downloads`
- `/support`
- `/licensing`

## Status

- Status Components Model: `blueprint_ready`
