# Public Portal Support Readiness Model

## Zweck

Dieses Dokument beschreibt die sichtbare Support-Readiness-Schicht des Portals.

## Sichtbare Signale

- `support_contact`
- `support_email`
- `support_readiness_state`

## Grenzen

- keine externe Zustellung
- kein Kontaktformular
- keine Ticketing- oder Helpdesk-Logik
- keine versteckte Operator-Aktion

## Verwendung im Portal

- `/support`
- Readiness-Summary-Komponente

## Status

- Support-Readiness-Modell: `blueprint_ready`
- echte Support-Zustellung: `approval_required`
