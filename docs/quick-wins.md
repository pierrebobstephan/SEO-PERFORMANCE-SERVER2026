# Quick Wins

Diese Punkte sind als risikoarme, spaeter moegliche Sofortmassnahmen priorisiert, werden hier aber nicht angewendet. Der primaere Umsetzungspfad ist jetzt WordPress-Plugin-basiert.

## 1. Homepage-Meta-Description plugin-seitig auf vollwertige Snippet-Laenge bringen

Umsetzungszone: `WordPress plugin primary path`

Warum jetzt:
Die aktuelle Description hat nur `40` Zeichen und ist ein enger, gut messbarer Scope.

Erfolg waere messbar an:
Meta-Description-Laenge im Bereich von etwa `140-160` Zeichen bei gleichbleibender Themenklarheit.

## 2. Eine einzige primaere H1 auf der Homepage herstellen

Umsetzungszone: `WordPress plugin primary path`

Warum jetzt:
Der aktuelle Lauf zeigt `H1 count: 2`, was sich plugin-seitig mit engem Homepage-Scope vorbereiten laesst.

Erfolg waere messbar an:
`H1 count: 1` im naechsten validierten Lauf.

## 3. Nicht essenzielles Homepage-Markup plugin-seitig abbauen

Umsetzungszone: `WordPress plugin primary path`

Warum jetzt:
`183759 bytes` HTML sind fuer eine Startseite ein sichtbarer Hebel.

Erfolg waere messbar an:
Sinkendem `html_bytes`-Trend ohne Layout- oder Inhaltsregression.

## 4. Strukturbezogene SEO-Signale auf der Homepage plugin-seitig ordnen

Umsetzungszone: `WordPress plugin primary path`

Warum jetzt:
Description, H1 und hohe HTML-Menge deuten zusammen auf verbesserbare Homepage-Semantik.

Erfolg waere messbar an:
Konsistenteren Signalen fuer H1, Description, Title und Markup ohne neue Seiteneffekte.

## 5. Cloudflare-Kompression spaeter als Sekundaerpfad pruefen

Umsetzungszone: `Cloudflare secondary path`

Warum nicht zuerst:
Kompression bleibt sinnvoll, ist aber nicht mehr der primaere erste Umsetzungspfad vor WordPress-Plugin-Massnahmen.

Erfolg waere messbar an:
Vorhandenem `Content-Encoding` im Report und stabiler oder besserer `response_ms`.
